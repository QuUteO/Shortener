# GoShortener

Высокопроизводительный сервис сокращения URL-адресов, построенный на микросервисной архитектуре (или в виде монолита) с использованием кэширования в оперативной памяти. Сервис позволяет мгновенно преобразовывать длинные ссылки в короткие токены, осуществлять быстрый редирект с субсекундной задержкой и собирать базовую аналитику переходов.

---

## Технологический стек

* **Язык разработки:** Go (Golang)
* **Веб-фреймворк:** Gin Gonic (высокопроизводительный HTTP API роутер)
* **База данных:** PostgreSQL (надежное хранилище оригинальных URL и статистики)
* **Кэширование:** Redis (для мгновенного резолвинга популярных ссылок в обход СУБД)
* **Алгоритм сокращения:** Base62 (генерация уникальных, визуально приятных и коротких идентификаторов)
* **Логирование:** Zap Logger (структурированные логи для продакшна)

---

## Архитектура и логика работы

Для достижения максимальной пропускной способности (RPS) на чтение реализована следующая схема:

```text
[Клиент] ──(Запрос: /r/{short_code})──> [Go App (Gin)]
                                              │
                    ┌─────────────────────────┴────────────────────────┐
             [Проверка в Redis]                                 [Мимо кэша?]
                    │                                                  │
         (Есть в кэше -> 301 Редирект)                       [Запрос в PostgreSQL]
                                                                       │
                                                            (Запись в Redis + Редирект)

```

1. **Генерация кода:** При создании короткой ссылки сервис использует автоинкрементный ID из PostgreSQL, конвертируя его в систему счисления **Base62** (символы `a-z`, `A-Z`, `0-9`). Это исключает коллизии и гарантирует минимальную длину ссылки.
2. **Кэширование редиректов:** При первом переходе по короткой ссылке её значение поднимается из PostgreSQL в Redis с настраиваемым TTL (например, 24 часа). Все последующие переходы обрабатываются в памяти за доли миллисекунды.
3. **Асинхронная аналитика:** Статистика кликов (User-Agent, IP, время) может буферизоваться в памяти или Redis и сбрасываться в PostgreSQL пачками, чтобы не блокировать поток редиректа.

## Настройка конфигурации (`config.yaml`)

Перед запуском приложения убедитесь, что конфигурационный файл заполнен актуальными данными для подключения к БД и Redis.

```yaml
env: #local dev prod
  env: local

postgres:
  host: localhost
  port: 5432
  user: user
  password: user
  database: shortener
  database_dsn: postgres://user:user@localhost:5432/shortener

redis:
  address: localhost:6379
  password:
  attempts: 3
  backoff: 2

http:
  address: localhost:8080
```

## Подготовка окружения перед запуском

Выполните следующие шаги последовательно, чтобы подготовить проект к работе:

### 1. Скачивание Go-зависимостей

Убедитесь, что все внешние библиотеки (драйверы баз данных, роутер, логер) загружены:

```bash
go mod tidy
go mod download

```

### 2. Запуск инфраструктуры (PostgreSQL + Redis)

Запустите контейнеры базы данных и кэша в фоновом режиме через Docker Compose:

```bash
docker-compose up -d

```

*Убедитесь, что порты `5432` и `6379` не заняты другими локальными сервисами.*

### 3. Применение миграций базы данных

Для создания таблиц `urls` и `clicks` примените миграции с помощью утилиты `goose` или `golang-migrate`:

```bash
goose -dir migrations postgres "postgres://user:user_password@localhost:5432/shortener_db?sslmode=disable" up
```

### 4. Проверка доступности портов

Перед стартом Go-сервиса проверьте, что инфраструктура готова принимать соединения:

```bash
# Проверить Postgres
nc -zv localhost 5432

# Проверить Redis
nc -zv localhost 6379
```

## Запуск проекта

Из корневой директории проекта выполните команду для запуска HTTP-сервера:

```bash
go run ./cmd/main.go
```

После этого API начнет принимать запросы по адресу `http://localhost:8080`.

## Спецификация API (Эндпоинты)

### 1. Сократить ссылку

* **URL:** `/api/v1/shorten`
* **Метод:** `POST`
* **Тело запроса (JSON):**

```json
{
  "long_url": "https://github.com/gin-gonic/gin/blob/master/README.md",
  "ttl_days": 30
}

```

* **Успешный ответ (201 Created):**

```json
{
  "short_code": "x7B2a",
  "short_url": "http://localhost:8080/r/x7B2a",
}

```

### 2. Редирект по короткому коду

* **URL:** `/r/:short_code`
* **Метод:** `GET`
* **Описание:** Автоматически перенаправляет пользователя (301 Moved Permanently) на оригинальный длинный URL-адрес. В процессе перенаправления в базу данных асинхронно записывается статистика перехода.

### 3. Получить аналитику по ссылке

* **URL:** `/api/v1/analytics/:short_code`
* **Метод:** `GET`
* **Успешный ответ (200 OK):**

```json
{
  "short_code": "x7B2a",
  "long_url": "https://github.com/gin-gonic/gin/blob/master/README.md",
  "clicks_count": 142,
  "created_at": "2026-07-14T11:30:00Z",
  "last_clicked_at": "2026-07-14T11:32:00Z"
}

```
